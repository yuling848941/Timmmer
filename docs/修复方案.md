### 2.1 修复 `UpdateDisplay` 中字体泄漏

**`timer_ui.c`** `UpdateDisplay` 函数末尾，在 `SelectObject(hdc, oldFont)` 之后添加：

```c
// 恢复原字体
if (oldFont) {
    SelectObject(hdc, oldFont);
}

// 删除创建的字体对象（仅删除非缓存字体）
// 注意：GetTimerFont/GetMonospaceFont/GetDefaultTimerFont 返回的字体
// 如果是从缓存获取的，不应在此删除；如果是新创建的，需要跟踪
// 最安全的做法：这些函数返回的字体由调用者负责释放
// 当前实现中，字体由 timer_fonts.c 的缓存系统管理，不应手动 DeleteObject
// 但 CalculateFontSize 中的测试字体必须删除（见 2.2）
```

**实际修复**: 检查 `GetTimerFont` / `GetMonospaceFont` / `GetDefaultTimerFont` 的实现。
- 如果这些函数返回缓存字体 → 不需要删除
- 如果每次调用都 `CreateFont` → 需要在 `UpdateDisplay` 末尾 `DeleteObject(hFont)`

根据代码分析，这些函数走缓存系统，**`UpdateDisplay` 中不需要额外删除**。但需要确认缓存不会无限增长。

### 2.2 修复 `CalculateFontSize` 中测试字体泄漏

**`timer_ui.c`** `CalculateFontSize` 函数，二分查找循环内：

```c
// 旧代码（约第 103-127 行）
while (minSize <= maxSize) {
    int testSize = (minSize + maxSize) / 2;
    HFONT testFont = GetMonospaceFont(testSize);
    if (!testFont) {
        testFont = GetDefaultTimerFont(testSize);
    }
    if (testFont) {
        HFONT oldFont = (HFONT)SelectObject(hdc, testFont);
        SIZE textSize;
        GetTextExtentPointA(hdc, testStr, strlen(testStr), &textSize);
        SelectObject(hdc, oldFont);
        // ... 判断逻辑
    }
}
```

**修改为**：
```c
while (minSize <= maxSize) {
    int testSize = (minSize + maxSize) / 2;
    HFONT testFont = GetMonospaceFont(testSize);
    BOOL needDelete = FALSE;
    if (!testFont) {
        testFont = GetDefaultTimerFont(testSize);
        needDelete = TRUE;  // 标记需要释放
    }
    
    if (testFont) {
        HFONT oldFont = (HFONT)SelectObject(hdc, testFont);
        SIZE textSize;
        GetTextExtentPointA(hdc, testStr, strlen(testStr), &textSize);
        SelectObject(hdc, oldFont);
        
        if (needDelete) {
            DeleteObject(testFont);
        }
        
        if (textSize.cx <= availableWidth && textSize.cy <= availableHeight) {
            bestSize = testSize;
            minSize = testSize + 1;
        } else {
            maxSize = testSize - 1;
        }
    } else {
        break;
    }
}
```

## 第 3 步：修复 GetTickCount 回绕问题

**影响模块**: `timer_core.c`, `timer_ui.c`
**风险等级**: 低（使用 64 位 API 替代 32 位）

### 3.1 替换 `GetTickCount` → `GetTickCount64`

| 文件 | 行号 | 旧代码 | 新代码 |
|------|------|--------|--------|
| `timer_core.c` | 65 | `g_timerState.startTime = GetTickCount()` | `g_timerState.startTime = (DWORD)GetTickCount64()` |
| `timer_core.c` | 145 | `DWORD currentTime = GetTickCount()` | `DWORD currentTime = (DWORD)GetTickCount64()` |
| `timer_ui.c` | 204 | `DWORD currentTime = GetTickCount()` | `DWORD currentTime = (DWORD)GetTickCount64()` |
| `timer_ui.c` | 250 | `g_timerState.startTime = GetTickCount()` | `g_timerState.startTime = (DWORD)GetTickCount64()` |
| `timer_ui.c` | 287 | `DWORD currentTime = GetTickCount()` | `DWORD currentTime = (DWORD)GetTickCount64()` |

### 3.2 修改 `timer_types.h` 中 `startTime` 类型

**`timer_types.h`** 第 221 行：
```c
// 旧
DWORD startTime;
// 新
DWORD64 startTime;
```

> **注意**: 如果改为 `DWORD64`，需要同步修改所有使用 `startTime` 的地方。
> **更保守的方案**: 保持 `DWORD` 类型不变，仅将 `GetTickCount()` 替换为 `GetTickCount64()` 后强制转换。
> 考虑到 `DWORD` 回绕周期为 49.7 天，而 Timer 是桌面应用（通常不会连续运行 49 天），
> **推荐保守方案**：仅替换 API 调用，不修改类型。

### 3.3 添加回绕安全计算

**`timer_ui.c`** `TimerTick` 函数中时间差计算：

```c
// 旧
DWORD elapsed = currentTime - g_timerState.startTime;

// 新（处理回绕）
DWORD elapsed = currentTime - g_timerState.startTime;
if (currentTime < g_timerState.startTime) {
    // GetTickCount 回绕发生
    elapsed = (MAXDWORD - g_timerState.startTime) + currentTime + 1;
}
```

## 第 4 步：修复配置文件路径问题

**影响模块**: `timer_config.c`
**风险等级**: 中（改变配置文件位置，首次运行会丢失旧配置）

### 4.1 创建统一的配置路径获取函数

**`timer_config.c`** 顶部添加辅助函数：

```c
// 获取用户配置目录路径
static BOOL GetUserConfigPath(wchar_t* outPath, DWORD maxLen) {
    // 优先使用 USERPROFILE 环境变量
    DWORD result = GetEnvironmentVariableW(L"USERPROFILE", outPath, maxLen);
    if (result > 0 && result < maxLen) {
        return TRUE;
    }
    
    // 回退：使用 SHGetFolderPath
    if (SUCCEEDED(SHGetFolderPathW(NULL, CSIDL_PROFILE, NULL, 0, outPath))) {
        return TRUE;
    }
    
    // 最后回退：当前目录
    result = GetCurrentDirectoryW(maxLen, outPath);
    return (result > 0 && result < maxLen);
}

// 构建完整配置文件路径
static void BuildConfigPath(wchar_t* outPath, DWORD maxLen) {
    GetUserConfigPath(outPath, maxLen);
    // 确保路径以反斜杠结尾
    size_t len = wcslen(outPath);
    if (len > 0 && outPath[len - 1] != L'\\') {
        wcscat_s(outPath, maxLen, L"\\");
    }
    wcscat_s(outPath, maxLen, CONFIG_FILE_PATH);
}
```
