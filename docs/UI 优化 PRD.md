## 1. 项目概述

### 1.1 产品背景
Timmer 是一个基于 Win32 API 和 GDI 开发的 Windows 桌面计时器应用程序。当前 UI 样式使用原生 Windows 控件，视觉效果较为传统，缺乏现代感。

### 1.2 优化目标
- 提升对话框/设置面板的视觉现代化程度
- 改善用户交互体验
- 保持轻量级特性（不引入重型框架）
- 兼容 Windows 10/11

### 1.3 技术约束
- 语言：纯 C 语言
- UI 框架：Win32 API / GDI / DWM
- 不允许引入 C++ 依赖或第三方 UI 库
- 保持单可执行文件部署

### 1.4 范围说明
**主计时器窗口**：当前功能满足使用需求，本阶段不纳入优化范围（除非后续提出新需求）
**聚焦范围**：设置对话框、菜单、系统托盘

### 2.2 主要问题汇总

#### 视觉层面
1. **颜色系统** - 使用系统默认灰色 (`COLOR_BTNFACE`)，缺乏品牌色
2. **控件样式** - 标准矩形按钮，无圆角、无阴影、无悬停效果
3. **布局设计** - 硬编码坐标，缺乏统一间距规范
4. **字体排印** - 默认系统字体，无层次结构
5. **视觉反馈** - 缺少状态指示、加载动画、过渡效果

#### 交互层面
1. **菜单** - 无法自定义，无图标，密集排列
2. **按钮** - 点击反馈仅限系统默认下沉效果
3. **设置** - 实时预览有限

#### 技术层面
1. **DPI 缩放** - 未适配高 DPI 显示器
2. **深色模式** - 不支持 Windows 深色主题
3. **无障碍** - 未考虑屏幕阅读器等辅助功能

### 4.2 技术选型对比

| 方案 | 优点 | 缺点 | 推荐度 |
|-----|------|------|--------|
| UXTheme API | 系统原生，兼容性好 | 自定义程度有限 | ★★★★ |
| 自绘控件 | 完全可控 | 工作量大 | ★★★ |
| DWM 扩展 | 现代效果好 | 仅 Win10/11 | ★★★★ |
| GDI+ | 绘图能力强 | 性能略低 | ★★ |

### 4.3 推荐技术栈

```
- 窗口装饰：DWM API (DwmSetWindowAttribute)
- 控件主题：UXTheme (SetWindowTheme)
- 自定义绘制：GDI + OwnerDraw
- 动画效果：SetTimer + 双缓冲
- 深色模式：DwmSetPreferredAppMode
```
